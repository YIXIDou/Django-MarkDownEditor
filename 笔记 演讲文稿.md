#### **3. 核心技术架构：深入 Django MTV (Core Architecture)**

* *演讲逻辑：展示专业知识，客观介绍技术栈。*
* **技术选型**：
* **后端**：Python 3.12 + Django 5.x（利用其“Batteries Included”特性快速构建安全后端）。
* **数据库**：SQLite（开发阶段轻量级存储）。


* **核心设计模式：MTV (Model-Template-View)**：
* **M (Model 数据层)**：
* 负责数据建模与数据库交互。定义了 `Post` (文章)、`User` (用户) 等模型，利用 Django ORM 屏蔽 SQL 细节。


* **T (Template 表现层)**：
* 负责页面展示。使用 DTL (Django Template Language) 动态渲染 HTML，将数据注入前端页面。


* **V (View 业务层)**：
* 系统的“大脑”。接收 URL 请求，调度 Model 获取数据，并选择 Template 进行渲染响应。

---
> “在技术选型上，我选择了 Python 3.12 搭配 Django 5.x 框架。Django 号称‘包含电池’的 Web 框架，也就是他基本上什么都有，它内置了强大的安全性支持和管理后台，非常适合快速构建严谨的内容管理系统。
> 本项目的核心架构遵循了经典的 MTV 设计模式，这也是我在开发过程中钻研最深的部分：
> 首先是 Model 数据层，Django 的 models.py 文件中定义了文章和用户的数据结构，利用 Django 的 ORM 系统，我不需要手写复杂的 SQL 语句就能操作数据库。
> 其次是 Template 表现层，Django 对 HTML 进行了重构，利用 Django 模板语言，我们可以实现页面的动态渲染和组件复用，比如导航栏的统一样式。
> 最后是 View 业务层，它是系统的指挥官。在 views.py 中，Django 的处理逻辑，先接收用户的 URL 请求，去数据库拿数据，再塞给前端页面，完成了整个闭环。
> 通过对 MTV 模式的实践，我们可以理解了一个 Web 请求从发出到响应的完整生命周期。”
---

#### **4. 系统架构详解：前端与后端 (System Overview)**

* *演讲逻辑：体现全栈开发的全局视野。*
* **后端架构 (Backend)**：
* **路由分发 (URLs)**：设计 RESTful 风格路径，精准导向业务逻辑。
* **业务逻辑 (Views)**：采用 **Class-Based Views (类视图)** 实现标准 CRUD 流程，代码复用率高。
* **权限控制 (Auth)**：集成 Mixins (`LoginRequiredMixin`, `UserPassesTestMixin`)，实现细粒度权限管理（如：仅作者可修改自己的文章）。


* **前端架构 (Frontend)**：
* **响应式布局**：基于 Bootstrap 4 (via `crispy-forms`) 实现移动端与 PC 端适配。
* **交互逻辑**：集成 JavaScript 库处理 Markdown 的实时渲染与图片上传。


---
>“接下来，我将从全栈的角度，详细拆解本系统的架构设计。
>在后端部分，我采用了 Django 推荐的‘类视图’（Class-Based Views）开发模式。相比于传统的函数式编程，类视图让我的代码结构更加清晰，复用性极高。同时，我利用 Mixin 混入类技术，实现了严格的权限控制，确保比如‘修改文章’这样的操作，只有文章的作者本人才能执行，保障了系统的安全性。
>在前端部分，为了不让界面显得简陋，我引入了 Bootstrap 4 响应式框架。这意味着我的博客不仅在电脑上好看，在手机和平板上也能自动适配布局。此外，我还通过 JavaScript 增强了页面的交互性，特别是在 Markdown 编辑器的实现上，前端起到了关键的实时渲染作用。
>通过前后端的紧密协同，系统实现了从数据处理到用户界面的无缝衔接。”

#### **5. 核心创新与改进：Markdown 编辑器 (Innovation)**
